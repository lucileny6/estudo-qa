# Estudo de QA

## Bug, Defeito e Falha

Defeito é um erro presente no código ou na lógica do sistema.

Bug é quando esse defeito é encontrado durante os testes e registrado pela equipe de QA.

Falha é quando o sistema apresenta um comportamento incorreto para o usuário final devido a um defeito no software.

---

## Severidade x Prioridade

Severidade representa o impacto técnico que um problema causa no sistema.

Prioridade indica a urgência com que o problema precisa ser corrigido.

Um bug pode ter baixa severidade mas alta prioridade, dependendo do impacto para o negócio.

---

## Tipos de Testes

Teste Unitário: verifica partes pequenas do código, como funções ou métodos.

Teste de Integração: verifica se diferentes módulos do sistema funcionam corretamente juntos.

Teste de Sistema: avalia o funcionamento completo do sistema.

Teste de Aceitação: valida se o sistema atende aos requisitos definidos pelo cliente.
